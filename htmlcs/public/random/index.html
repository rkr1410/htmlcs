Ye olde HTML/CSS learning checklist

HTML

  • link
    • Describes a relation between the current document and an external resource. Typically this is a stylesheet, but can be a favicon or main screen application icon for a mobile app.

    HTML

        <link href="https://fonts.googleapis.com/css2?family=Grenze+Gotisch" rel="stylesheet">

    CSS

        h3 {
        font-family: "Grenze Gotisch", cursive;
        font-size: 3rem;
        text-align: center;
        text-shadow: crimson 1px 0 7px;
    }
    Note: in this case, instead of <link>ing the font, we can @import it directly into the CSS: 
        @import url('https://fonts.googleapis.com/css2?family=Grenze+Gotisch');

  • Tables
         
    • A maxed out table is: <caption>, <colgroup>, <thead>, <tbody> and a <tfoot>.
    Colgroup contains <col> elements, which can be styled separately, and can have a span attribute.

    The old-school table attributes are deprecated:
    • align
      • left, right, center: use margin: auto trick for either or both.
    • border
      • Just use CSS border.
    • cellpadding
      • Apply border-collapse: collapse to the <table> and padding to the <td> elements.
    • cellspacing
      • Remove border-collapse: collapse from the <table> and add border-spacing instead.

  • variables (CSS 3.0)
    • There's a :root pseudoclass, matching the document tree root - in case of html this is equivalent to <html>, except has greater specificity. It's useful to define global CSS variables there: 
        :root {
        --primary-color: black;
    }

    .code {
        color: var(--primary-color);
    }

  • margin
    • Margins values (hover the 'o' to see):
    5px - all of them
    o
    5px 15px - top&down left&right
    o
    15px 15px -15px - top left&right bottom
    o
    0px 10px 20px 30px - top right bottom left
    o

    You can center an element using margin: 0 auto, provided the box is smaller than 100%. It won't work on inlined elements, since these are treated more or less like text.
    Note: margin: auto does the trick, because usually top and bottom are calculated to zero. MDN-canonical way seems to be always setting top/down, too (even if these are supposed to evaulate to zero) with margin: 0 auto
    width: 50%; margin: 10px auto;
    And then content can be centered just with text-align: center
    width: 60%; margin: 10px auto; text-align: center;

  • sizing
    • em
    Parent em - e.g. if you have a div > p and the div has a font-size of 15px, then setting p to 2em will result on 30px font size. If nothing is set, it will use some default value (em = ~16px)

    rem
    Root em. Less tricky than em. Say you have a WordPress widgets made up from separate php files - you don't know the em value at given moment. Changing something in one place might blow up elsewhere.
    Or you have a 0.75em font-size in your ul's, then nesting another inside will result in 0.75*0.75 font size.

    vh, vw
    Percentage of viewport height and width. Seems to be useful for setting margins/padding for top-level components.

    ch
    What's this, exactly? Also, are there any other units?

    responsive design
    Learn it, live it.

    So, some rules of thumb could be:
    - use vh/vw for html/body
    - rem for component heights
    - percent for widths
