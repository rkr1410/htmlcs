<!--DOCTYPE html>

spec

<!DOCTYPE html> is a required preamble at the top of an HTML5 document. It prevents the browser from going into quirks mode (a page rendering flow which emulates nonstandard behavior in old browsers).

Although the specification says it's case-insensitive, the provided example is a widely used convention. W3 Polyglot Markup requires the 'DOCTYPE' string to be uppercase, and suggests 'html' to be lowercase.

This is not a tag - so it doesn't have an end tag

<html lang="en">

spec

HTML document root. Specifying a lang attirbute will help speech synthesis in determining pronounciation rules, and translation popups might just stop pestering the user.

This element might be important for absolute positioning. If an element has position: absolute; and none of it's parent specifically set position: relative;, then the absolute element will be positioned relative to html (and not body as mentioned by some tutorials)

html > <head>

spec

May contain:

  • title
  • base
  • meta
  • link
  • style

html > head > <title>

spec

First element in head section. Spec states it should be meaningful out of context. Can be omitted when provided by higher-level context (e.g. email subject, when html is used for the body).

html > head > <base>

spec

Similar to title, if it's present there can be only one. Specifies the base URL for the document and default link navigation behavior.

base href="http://abc.com/index.html" target="_blank" will make clicking on a link a href="a.html" open http://abc.com/a.html in a new browser tab.

html > head > <link rel href>

spec

Describes a relation between the current document and an external resource. Typically this is a stylesheet, but can be a google font, favicon or main screen application icon for a mobile app. So, in head section:

link href="https://fonts.googleapis.com/css2?family=..." rel="stylesheet"

In this specific case (getting a google font) we could just go to CSS and @import url('font.url');
